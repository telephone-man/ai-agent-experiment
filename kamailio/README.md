## Kamailio Foundation Notes

This directory keeps the working Kamailio baseline needed by the local
voice-agent demo:

- `kamailio.cfg` remains the top-level routing entrypoint with `request_route` visible.
- `config/*.cfg` holds grouped include files for modules, registration flow, outbound routing, and dialog/media handling.
- `kamailio-entrypoint.sh` renders runtime placeholders, waits for Postgres,
  and creates the Kamailio schema with `kamdbctl create` when the database is
  absent.

### Local topology

The config is intentionally single-node: one Kamailio instance, one RTPengine,
and one FreeSWITCH target at `sip:freeswitch:5080`. Registrations are stored
locally, and calls to unregistered peers fail immediately with `480`.

There is no checked-in seed SQL file. Database schema creation is handled at
container startup by `kamdbctl` using local credentials generated by
`./setup.sh`.
